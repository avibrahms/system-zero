# Business — Commercialization

**Purpose:** Commercialize the engine first, then use product-specific distribution only when a product has real pull or an audience-specific reason.

**Full context:** See `README.md` in this folder.

## Strategy

Default strategy:
- sell autonomous execution infrastructure / `System0` / engine capabilities
- convert LinkedIn attention into design-partner and investor conversations
- use internal products as proof, not as a scattered catalog
- pursue product-specific partner motions only when there is concrete demand

Channel partners still matter, but engine-first positioning takes priority over Lucid-first outreach.

- `business--offer-execution` owns the live warm-offer lane and keeps the current business revenue path executable.
- Decision Desk private rescue chain is historical context only. The active warm lane is the System Zero design-partner pack at `modules/business/outreach-drafts/system-zero-design-partner-first-5-2026-04-09.md`.
